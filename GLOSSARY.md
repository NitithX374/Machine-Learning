## 2. การเทรนโมเดล — วงจรหลักของทุกอย่าง

> วงจรเดียวที่ใช้ทั้งวงการ: **forward → loss → backprop → update → วนซ้ำ**

| ศัพท์ | ความหมาย | เล่ม |
|---|---|---|
| **Model** | ฟังก์ชันที่มี parameter ปรับได้ ใช้แปลง input เป็นคำทำนาย | ทุกเล่ม |
| **Parameter / Weight** | ตัวเลขภายในโมเดลที่ถูก "เรียนรู้" ระหว่างเทรน | 1 |
| **Bias** | ค่าเลื่อนฐานของ neuron (parameter อีกชนิด) | 1 |
| **Forward pass** | การคำนวณคำตอบของโมเดลจาก input (ไล่จากต้นไปปลาย) | 1 |
| **Loss / Cost function** | ตัววัดว่าคำทำนายผิดจากเฉลยแค่ไหน (ยิ่งต่ำยิ่งดี) | 1 |
| **MSE** (Mean Squared Error) | loss แบบกำลังสองของส่วนต่าง — ใช้กับงานทำนายค่าต่อเนื่อง | 1 |
| **Cross-Entropy** | loss มาตรฐานของงานจำแนกประเภท (classification) | 1 |
| **Gradient** | ความชันของ loss เทียบกับ parameter — บอกทิศที่ควรปรับ | 1 |
| **Backpropagation** | อัลกอริทึมหา gradient ของทุก parameter ด้วย **chain rule** (ไล่ย้อนจากปลายมาต้น) | 1 |
| **Chain rule** | กฎลูกโซ่ของแคลคูลัส — หัวใจของ backprop | 1 |
| **Gradient Descent** | การปรับ parameter "สวนทาง" gradient เพื่อลด loss ทีละก้าว | 1 |
| **Learning rate** | ขนาดก้าวของการปรับ parameter (ใหญ่ไป = เด้ง, เล็กไป = ช้า) | 1 |
| **Optimizer** | ตัวจัดการการ update เช่น SGD, **Adam** (ฉลาดกว่า SGD เปล่า) | 1 |
| **Epoch** | การวนดูข้อมูลทั้งชุดครบ 1 รอบ | 1 |
| **Batch** | กลุ่มย่อยของข้อมูลที่ป้อนพร้อมกัน 1 ครั้ง | 1 |
| **Hyperparameter** | ค่าที่เราตั้งเอง (ไม่ได้เรียนรู้) เช่น learning rate, จำนวนชั้น | 1 |
| **Reproducibility / Seed** | การ fix ค่าสุ่มให้รันซ้ำแล้วได้ผลเดิม | ทุกเล่ม |

## 3. Neural Network & Deep Learning

| ศัพท์ | ความหมาย | เล่ม |
|---|---|---|
| **Neuron** | หน่วยคำนวณพื้นฐาน: ผลรวมถ่วงน้ำหนัก + activation | 1 |
| **Layer** | ชั้นของ neuron ที่ทำงานพร้อมกัน | 1 |
| **Hidden layer** | ชั้นกลางระหว่าง input กับ output (ที่ทำให้แก้ปัญหาซับซ้อนได้) | 1 |
| **MLP** (Multi-Layer Perceptron) | neural network แบบชั้นต่อชั้นพื้นฐานที่สุด | 1 |
| **Activation function** | ฟังก์ชันไม่เชิงเส้นที่ทำให้โมเดลเรียนรูปแบบซับซ้อนได้ | 1 |
| **Sigmoid** | activation บีบค่าเป็น 0–1 (ตีความเป็นความน่าจะเป็น) | 1 |
| **ReLU** | activation มาตรฐานยุคนี้: ค่าลบ→0, ค่าบวก→คงเดิม | 1 |
| **Softmax** | แปลงคะแนนเป็นความน่าจะเป็นหลายคลาสที่รวมกันได้ 1 | 1 |
| **Vectorization** | คิดทั้งชั้น/ทั้ง batch เป็นการคูณ matrix ครั้งเดียว (เหตุผลที่ใช้ GPU) | 1 |
| **CNN** (Convolutional NN) | สถาปัตยกรรมสำหรับรูปภาพ — ใช้ filter สแกนหา pattern | 1 |
| **Convolution / Filter (Kernel)** | ตัวตรวจจับ pattern เล็ก ๆ ที่เลื่อนสแกนทั่วภาพ (เรียนรู้ค่าเอง) | 1 |
| **Pooling** | การย่อภาพโดยเก็บสัญญาณเด่น (เช่น MaxPooling) | 1 |
| **Embedding** | การแปลงสิ่งของ (token/node) เป็นเวกเตอร์ที่มีความหมาย | 1, 2 |
| **Logits** | คะแนนดิบก่อนเข้า softmax | 1, 2 |

## 4. การวัดผล & ปัญหาที่พบบ่อย

| ศัพท์ | ความหมาย | เล่ม |
|---|---|---|
| **Classification** | งานทำนาย "ประเภท" (เลขอะไร, botnet หรือไม่) | 1, 3 |
| **Regression** | งานทำนาย "ค่าต่อเนื่อง" (ราคา, อุณหภูมิ) | 1 |
| **Accuracy** | สัดส่วนที่ทายถูก — ระวัง! หลอกตาเมื่อข้อมูลไม่สมดุล | 1, 3 |
| **Precision** | ในของที่เตือนว่า "ใช่" จริงกี่ส่วน (สำคัญเมื่อเตือนผิดแล้วแพง) | 3 |
| **Recall** | ในของจริงทั้งหมด จับได้กี่ส่วน (สำคัญเมื่อพลาดแล้วแพง) | 3 |
| **Confusion matrix** | ตารางสรุปว่าทายคลาสไหนเป็นคลาสไหน (เห็นคู่ที่สับสน) | 1, 3 |
| **Overfitting** | โมเดล "ท่องจำ" ข้อมูลเทรน (รวม noise) จนใช้กับของใหม่ไม่ได้ | 1 |
| **Underfitting** | โมเดลง่ายเกินไป จับ pattern ไม่ได้แม้แต่ในข้อมูลเทรน | 1 |
| **Generalization** | ความสามารถทำงานดีกับข้อมูล**ใหม่ที่ไม่เคยเห็น** (เป้าหมายแท้จริง) | 1 |
| **Regularization** | เทคนิคกัน overfitting (จำกัด "ความจุ" ของโมเดล) | 1 |
| **Dropout** | สุ่มปิด neuron ระหว่างเทรน เพื่อกัน overfitting | 1 |
| **Early stopping** | หยุดเทรนตอน validation loss เริ่มแย่ลง | 1 |
| **Class imbalance** | ข้อมูลที่คลาสหนึ่งมีน้อยมาก (ทำให้ accuracy หลอกตา) | 3 |
| **Normalization** | ปรับสเกล input ให้ใกล้เคียงกัน ช่วยให้เทรนเสถียร | 1, 3 |

## 5. LLM & การประมวลผลภาษา

| ศัพท์ | ความหมาย | เล่ม |
|---|---|---|
| **Token** | หน่วยย่อยของข้อความ (ตัวอักษร/subword/คำ) | 1, 2 |
| **Tokenizer** | ตัวตัดข้อความเป็น token แล้วแปลงเป็นตัวเลข | 2 |
| **BPE** (Byte Pair Encoding) | อัลกอริทึม tokenizer: รวมคู่ที่เจอบ่อยให้เป็น token เดียว | 2 |
| **Vocabulary (vocab)** | ชุด token ทั้งหมดที่โมเดลรู้จัก | 2 |
| **Attention / Self-Attention** | กลไกให้แต่ละ token "มอง" token อื่นเพื่อเข้าใจบริบท | 1 |
| **Q / K / V** (Query/Key/Value) | 3 เวกเตอร์ที่แต่ละ token สร้างเพื่อทำ attention | 1 |
| **Multi-Head Attention** | ทำ attention หลายชุดพร้อมกัน จับความสัมพันธ์หลายมุม | 1 |
| **Positional Encoding** | การบอกโมเดลว่า token อยู่ตำแหน่งไหน (attention ไม่สนลำดับเอง) | 1 |
| **Causal Mask** | ปิดไม่ให้ token "มองอนาคต" — หัวใจที่ทำให้เป็น LLM | 1 |
| **Transformer** | สถาปัตยกรรม attention + feed-forward ซ้อนกัน (แกนของ LLM) | 1 |
| **Pretraining** | เทรนขั้นแรกบนข้อความมหาศาล โจทย์ "ทาย token ถัดไป" | 2 |
| **Autoregressive** | สร้างผลทีละชิ้น แล้วป้อนกลับเข้าไปทำต่อ (generate ข้อความ) | 1, 2 |
| **Fine-tuning** | เอาโมเดล pretrained มาเทรนต่อบนงานเฉพาะ | 2 |
| **Transfer Learning** | การนำความรู้จากโมเดลที่เทรนแล้วมาต่อยอด | 2 |
| **Temperature** | ปรับความ "กล้าเสี่ยง" ตอน generate (ต่ำ=นิ่ง, สูง=หลากหลาย) | 2 |
| **Top-k / Top-p** | กลยุทธ์ sampling: เก็บแค่ token ที่น่าจะเป็นสูงสุด k ตัว / สะสมถึง p | 2 |
| **Greedy decoding** | เลือก token ที่น่าจะเป็นสูงสุดเสมอ | 2 |
| **KV Cache** | เก็บ Key/Value เดิมไว้ ไม่คำนวณซ้ำ → generate เร็วขึ้นมาก | 2 |
| **Context window** | จำนวน token สูงสุดที่โมเดลมองย้อนหลังได้ | 2 |
| **Catastrophic forgetting** | โมเดล "ลืม" ความรู้เดิมเมื่อ fine-tune แรงเกินไป | 2 |
| **RLHF** | ใช้ RL + ความชอบของคน ฝึกโมเดลให้ตอบแบบที่คนชอบ | 2, 4 |
| **Reward model** | โมเดลที่ทำนาย "คะแนนความพอใจของคน" ต่อคำตอบ | 4 |
| **PPO** | อัลกอริทึม policy gradient ที่ใช้ใน RLHF (พัฒนาจาก REINFORCE) | 4 |
| **RAG** | ดึงข้อมูลที่เกี่ยวข้องมาแปะใน prompt ก่อนให้โมเดลตอบ | 2 |

## 7. Reinforcement Learning

| ศัพท์ | ความหมาย | เล่ม |
|---|---|---|
| **Agent** | ตัวที่ตัดสินใจและเรียนรู้ (ผู้เล่น) | 4 |
| **Environment** | โลกที่ agent อยู่ ให้ state ใหม่และ reward | 4 |
| **State** | สถานการณ์ปัจจุบันที่ agent เห็น | 4 |
| **Action** | สิ่งที่ agent เลือกทำในแต่ละ state | 4 |
| **Reward** | ผลตอบแทนที่ environment ให้หลังทำ action (สัญญาณเดียวที่บอกว่าดี/แย่) | 4 |
| **Policy** | "นโยบาย" — จะทำ action อะไรในแต่ละ state | 4 |
| **Episode** | 1 รอบของเกม ตั้งแต่เริ่มจนจบ | 4 |
| **Q-value** | ค่าคาดหวังของ reward รวมระยะยาว ถ้าทำ action นี้ใน state นี้ | 4 |
| **Q-Learning** | อัลกอริทึม RL พื้นฐานที่เรียน Q-value (เก็บในตาราง) | 4 |
| **Bellman equation** | สมการอัปเดต Q: reward ตอนนี้ + ค่าที่ดีสุดของ state ถัดไป | 4 |
| **Discount factor (γ)** | ให้น้ำหนัก reward อนาคตแค่ไหน (0–1) | 4 |
| **Exploration vs Exploitation** | ชั่งใจระหว่าง "ลองของใหม่" กับ "ใช้ของที่รู้ว่าดี" | 4 |
| **Epsilon-greedy** | กลยุทธ์: สุ่มสำรวจด้วยโอกาส ε ที่เหลือเลือกตัวที่ดีสุด | 4 |
| **DQN** (Deep Q-Network) | ใช้ neural net ประมาณ Q-value แทนตาราง (state เยอะ ๆ) | 4 |
| **Experience Replay** | เก็บประสบการณ์ลง buffer แล้วสุ่มมาเทรน (ช่วยให้เสถียร) | 4 |
| **Target Network** | network สำเนาอัปเดตช้า ๆ ใช้คำนวณเป้าหมาย (กัน training พัง) | 4 |
| **Policy Gradient / REINFORCE** | เรียน policy โดยตรง: ดัน action ที่ได้ reward ดีให้น่าจะเป็นมากขึ้น | 4 |
| **Value-based vs Policy-based** | สองตระกูล RL: เรียน "ค่า" (DQN) vs เรียน "นโยบาย" (REINFORCE) | 4 |

## 8. Generative Models

| ศัพท์ | ความหมาย | เล่ม |
|---|---|---|
| **Generative vs Discriminative** | สร้างข้อมูลใหม่ ($p(x)$) vs จำแนกประเภท ($p(y\|x)$) | 5 |
| **Latent space** | ปริภูมิเวกเตอร์ย่อที่เข้ารหัส "แก่น" ของข้อมูล | 5 |
| **VAE** (Variational Autoencoder) | autoencoder ที่ latent เป็นการแจกแจง → สุ่มสร้างภาพใหม่ได้ | 5 |
| **Reparameterization trick** | เขียนการสุ่ม $z=\mu+\sigma\epsilon$ ให้ gradient ไหลผ่านได้ | 5 |
| **KL Divergence** | วัดความต่างของ 2 การแจกแจง (บังคับ latent ให้ใกล้ N(0,1)) | 5 |
| **GAN** | generator ปะทะ discriminator — แข่งกันจนวาดภาพเนียน | 5 |
| **Generator / Discriminator** | ตัวสร้างของปลอม / ตัวจับว่าจริงหรือปลอม | 5 |
| **Mode collapse** | generator วาดได้ไม่กี่แบบซ้ำ ๆ (ปัญหาคลาสสิกของ GAN) | 5 |
| **Diffusion Model** | เติม noise แล้วเรียน "ถอย noise" — เบื้องหลัง Stable Diffusion | 5 |
| **Forward / Reverse process** | เติม noise ทีละขั้น (กำหนดไว้) / ถอย noise (ต้องเรียน) | 5 |
| **DDPM** | Denoising Diffusion Probabilistic Model — สูตร diffusion มาตรฐาน | 5 |
| **U-Net** | สถาปัตยกรรมหด-ขยายพร้อม skip connection (ใช้ใน diffusion) | 5 |

## 9. Graph Neural Networks

| ศัพท์ | ความหมาย | เล่ม |
|---|---|---|
| **Graph** | ข้อมูลที่เป็นจุด (node) เชื่อมกันด้วยเส้น (edge) | 6 |
| **Node / Edge** | จุด (เช่น เครื่อง/คน) / เส้นเชื่อม (ความสัมพันธ์) | 6 |
| **Adjacency matrix** | matrix $N\times N$ บอกว่า node ไหนต่อกับ node ไหน | 6 |
| **GNN** (Graph Neural Network) | NN ที่ทำงานบนกราฟ ผ่านการให้ node คุยกับเพื่อนบ้าน | 6 |
| **Message Passing** | หัวใจ GNN: node ดูดข้อมูลจากเพื่อนบ้านมาอัปเดตตัวเอง | 6 |
| **GCN** (Graph Convolutional Network) | GNN แบบนิยมสุด: $H'=\sigma(\hat{A}HW)$ | 6 |
| **Node classification** | จำแนกประเภทของแต่ละ node (เช่น botnet หรือไม่) | 6 |
| **Link prediction** | ทำนายว่า 2 node ควรเชื่อมกันไหม (ระบบแนะนำ) | 6 |
| **Over-smoothing** | ซ้อนชั้น GNN มากไป ทุก node เหมือนกันหมด (จึงใช้แค่ 2–3 ชั้น) | 6 |

## 10. ML Pitfalls — ทำให้ถูก

| ศัพท์ | ความหมาย | เล่ม |
|---|---|---|
| **Data Leakage** | ข้อมูลที่ไม่ควรเห็นตอนเทรน "รั่ว" เข้าไป → ผลลวงตา (กับดักอันดับ 1) | 7 |
| **Target Leakage** | มี feature ที่จริง ๆ คำนวณจาก/รู้คำตอบ (รั่วชนิดร้ายแรง) | 7 |
| **Pipeline** | ห่อ preprocessing + โมเดลเข้าด้วยกัน เพื่อ fit จาก train เท่านั้น (กัน leakage) | 7 |
| **Cross-Validation (k-fold)** | แบ่งข้อมูลเป็น k ส่วน วนวัดแล้วเฉลี่ย → ตัวเลขนิ่งกว่า split เดียว | 7 |
| **Baseline** | โมเดลโง่ ๆ (เช่น ทายคลาสที่เยอะสุด) ที่ต้องเทียบด้วยเสมอ | 7 |
| **ROC-AUC** | วัดความสามารถ "จัดอันดับ" ของโมเดล (0.5=มั่ว, 1.0=สมบูรณ์) | 7 |
| **Calibration** | ความน่าจะเป็นที่โมเดลบอกตรงกับความถี่จริงไหม (เชื่อ "ความมั่นใจ" ได้ไหม) | 7 |
| **Brier score** | วัดคุณภาพของความน่าจะเป็นที่ทำนาย (ยิ่งต่ำยิ่งดี) | 7 |
| **Threshold** | ค่าตัดสิน 0/1 (default 0.5 ไม่เหมาะกับข้อมูลไม่สมดุล) | 7 |
| **Test-set overfitting** | จูน/เลือกโมเดลบน test ซ้ำ ๆ จนหลอกตัวเอง — test ต้องแตะครั้งเดียว | 7 |
